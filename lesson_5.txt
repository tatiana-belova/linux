Задание 1. Создание ссылок:
- создать файл file1 и наполнить его произвольным содержимым.
	echo "new_file" > file1
- скопировать файл file1 в file2.
	cat file1 > file2
- создать символическую ссылку file3 на file1.
	ln -s file1 file3
- создать жесткую ссылку file4 на file1.
	ln file1 file4
- посмотреть, какие айноды у файлов.
	ls -li
- удалить file1. 
	rm file1
Что стало с остальными созданными файлами? Попробовать вывести их содержимое на экран.
 	cat file2 file3 file4
	file2 не изменился (показывает наполнение - new_file)
  	cat file3  выводит No such file or directory (т.к. символическая ссылка ссылается на удаленный файл)
	file4 так же показывает наполнение - new_file, так как жесткая ссылка после удаления file1 продолжила работать.
- удалить все остальные файлы
	rm file2 file3 file4

Задание 2. Работа со ссылками:
- создать файл file1 и наполнить его произвольным содержимым.
 	echo "new_file" > file1
- создать символическую ссылку file1_link, указывающую на этот файл.
	ln -s file1 file1_link
- переместить ссылку в другую директорию.   Если не работает, как сделать так, чтобы работала?
    	mv file1_link ..
Работает ли она?
    	cat ../file1_link
  	cat: ../file1_link: No such file or directory (ссылка не работает)
Почему?
	Ссылка не работает, так как необходимо указать абсолютный путь к исходному файлу при создании ссылки:
  	 ln -s $(pwd)/file1 file1_link
    	mv file1_link ..
    	cd ..
    	cat file1_link
  	new_file

Задание 3. Права файлов:
- создать два произвольных файла.
	touch file1
	touch file2
- первому файлу присвоить права на чтение, запись для владельца и группы; на чтение для всех. Сделать это в численном и символьном виде.
	Присвоение права на чтение, запись для владельца и группы; на чтение для всех в численном виде - при помощи команды chmod 664 file1
	Присвоение права на чтение, запись для владельца и группы в символьном виде - при помощи команды chmod ug+rw file1; 
	на чтение для всех - chmod a+r file1
- второму файлу присвоить права на чтение, запись только для владельца. Сделать это в численном и символьном виде.
	Присвоение права на чтение, запись только для владельца в численном виде при помощи команды chmod 600 file2
   	Присвоение права на чтение, запись только для владельца в символьном виде при помощи команды chmod u+rw file2
