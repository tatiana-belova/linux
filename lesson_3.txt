Задание 1.
а) Чтобы создать пользователя user1, указав shell /bin/bash и создав домашний каталог, при помощи утилиты useradd,
мне понадобилась команда sudo useradd -m -s /bin/bash user1, где -m - это создание отдельного домашнего каталога для
user1, а -s - для указания shell.
b) Чтобы создать пользователя user2, указав shell /bin/bash и создав домашний каталог, используя утилиту adduser, я использовала
команду sudo adduser --home /users/new_user2 --shell /bin/bash user2, где --home используется для созадния домашнего
каталога, а --shell для указания shell.
c) Для удаления пользователя user1 вместе с домашним каталогом при помощи утилиты userdel, мне понадобилась команда
sudo userdel  --remove user1, где --remove удаляет пользователя вместе с домашним каталогом.
d) Для создания пользователя user1, с основной группой user2, я использовала команду sudo useradd -g user2 user1.
e) Для смены пароля пользователя user1 при помощи passwd, я воспользовалась командой sudo passwd user1.
f) Для смены пароля пользователя user2, используя usermod, мне понадобилась команда 
 sudo usermod -p $(openssl passwd new_password) user2. (openssl генерирует зашифрованый пароль)
Задание 2.
a) Чтобы создать группы group1, group2 с использованием groupadd я воспользовалась командами sudo groupadd group1 и
sudo groupadd group2.
b) Чтобы добавить user1 в группы group1, group2 разом при помощи одной команды usermod, не меняя его основной группы
я использовала команду sudo usermod -a -G group1,group2 user1, где -a добавит пользователя в дополнительную группы,
а -G для указания групп, в которые пользователя нужно добавить.
с) Чтобы создать группу user1 с GID=1516 и сделать ее основной для пользователя user1, я сначала использовала команду
sudo groupadd -g 1516 user1, где -g используется, чтобы новая группа пользовалась указанным GID, затем я применила
sudo usermod -g user1 user1, где -g принудительно назначает основную группу user1.
Задание 3.
Чтобы создать пользователя с правами суперпользователя, я воспользовалась командой:
useradd -ou 0 -g 0 superuser, где -ou 0 и -g 0 нужны, чтобы он входил в ту же группу и имел те же права, что и root.
passwd 1234567
Чтобы сделать так, чтобы sudo не требовал пароль для выполнения команд - с помощью команды sudo visudo открываем файл /etc/sudoers
находим в открывшемся файле нужного пользователя (в нашем случае - superuser) добавляем ALL=(ALL:ALL) NOPASSWD: ALL.
Задание 4.
Используя дополнительные материалы, выдать одному из созданных пользователей право на выполнение ряда команд,
требующих прав суперпользователя (команды выбираем на своё усмотрение).
Открываем файл sudoers при помощи команды sudoedit /etc/sudoers
Находим имя нужного пользователя и, в зависимости от требований, добавляем различные команды, например:
1. если мы хотим позволить пользователю, используя утилиту sudo, выполнять административные действия без ограничений,
мы добавим User2 ALL=(ALL) ALL
2. если мы хотим позволить пользователю с именем user2, используя sudo, добавлять учётные записи в системе, мы пропишем следующее:
user2 All= /usr/sbin/adduser (необходимо прописывать полный путь до команды).
